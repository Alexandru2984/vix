VixArena

VixArena is a minimalist real-time 2D multiplayer arena built with a C++ backend. Vix.cpp was installed and evaluated on this VPS, then a Boost.Beast production fallback was used because the available Vix HTTP listener did not bind localhost-only. The app keeps authoritative in-memory game state, fills solo arenas with server-side bots, accepts WebSocket input and abilities, broadcasts snapshots at 20 ticks/sec, and serves a plain browser canvas frontend with an objective HUD, minimap, edge target markers, score popups, quick pings, sound toggle, and touch controls.

Controls

  • WASD or arrow keys move the player.
  • Space uses dash.
  • Shift uses shield/phasing.
  • E uses magnet.
  • The touch joystick moves the player on mobile and touchscreen devices. Dragging directly on the arena also starts a floating joystick.
  • Ability buttons and quick pings work on desktop and touch devices.
  • When humans are connected, bots fill the arena up to four total players for solo practice.
  • Collect glowing orbs for instant points.
  • Complete Orb Run by collecting 3 orbs for a bonus.
  • Grab violet speed boosts for temporary faster movement.
  • Hold the green control zone for passive points. If multiple players contest it, scoring pauses.
  • Follow the objective HUD and edge markers to find nearby orbs, boosts, or the control zone.
  • Enter focuses chat.
  • Esc unfocuses chat.

HTTP endpoints

  • GET / game page.
  • GET /health health JSON with service status, player count, and uptime.
  • GET /api/state sanitized world state with player count, world size, obstacles, orbs, powerups, round state, and control zone.
  • GET /api/stats runtime counters, human/bot player counts, tick target, pickups, rounds, and control-zone points.
  • GET /api/leaderboard persistent leaderboard sorted by wins, best score, total score, and name.
  • GET /api/matches recent persisted match history.
  • GET /metrics Prometheus text metrics with player counts, WebSocket traffic, tick durations, rejected messages, pickups, quests, and rounds.
  • GET /stats public stats page with runtime counters, leaderboard, and match history.
  • GET /docs this page.

WebSocket

Public endpoint: wss://vix.micutu.com/ws. Browser clients automatically use wss:// when the page is served over HTTPS.

Validation and limits

  • Invalid JSON and unknown message types are rejected.
  • Display names are capped at 18 bytes and chat at 220 bytes.
  • WebSocket payloads are capped at 4096 bytes.
  • Input is throttled server-side and the arena caps active joined players.
  • Movement, abilities, cooldowns, speed boosts, orb pickup scoring, mini quest progress, round state, and control-zone scoring are server-authoritative.
  • Chat is throttled per connection and the last 50 messages are kept in memory.

Limitations of v1

  • Persistence covers leaderboard and match history, not full account identity.
  • No accounts, private rooms, or anti-abuse identity layer.
  • Snapshot deltas are JSON deltas rather than binary packets.
  • Mobile support is still v1-level; landscape-specific UI modes, haptics, and richer gesture controls are TODOs.
