VixArena

VixArena is a minimalist real-time 2D multiplayer arena built with a C++ backend. Vix.cpp was installed and evaluated on this VPS, then a Boost.Beast production fallback was used because the available Vix HTTP listener did not bind localhost-only. The app keeps authoritative in-memory game state, accepts WebSocket input, broadcasts snapshots at 20 ticks/sec, and serves a plain browser canvas frontend.

Controls

  • WASD or arrow keys move the player.
  • Collect glowing orbs for instant points.
  • Hold the green control zone for passive points.
  • Enter focuses chat.
  • Esc unfocuses chat.

HTTP endpoints

  • GET / game page.
  • GET /health health JSON with service status, player count, and uptime.
  • GET /api/state sanitized world state with player count, world size, obstacles, orbs, and control zone.
  • GET /api/stats runtime counters, tick target, pickups, and control-zone points.
  • GET /docs this page.

WebSocket

Public endpoint: wss://vix.micutu.com/ws. Browser clients automatically use wss:// when the page is served over HTTPS.

Validation and limits

  • Invalid JSON and unknown message types are rejected.
  • Display names are capped at 18 bytes and chat at 220 bytes.
  • WebSocket payloads are capped at 4096 bytes.
  • Movement, orb pickup scoring, and control-zone scoring are server-authoritative.
  • Chat is throttled per connection and the last 50 messages are kept in memory.

Limitations of v1

  • No persistence across restarts.
  • No accounts, private rooms, or anti-abuse identity layer.
  • Snapshots are full lightweight world snapshots rather than binary deltas.
  • Mobile controls are not implemented beyond responsive layout.
